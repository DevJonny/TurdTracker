## Codebase Patterns
- Build command: `DOTNET_CLI_USE_MSBUILD_SERVER=0 DOTNET_HOST_PATH=$(which dotnet) dotnet build` (required on macOS ARM64 to avoid task host error)
- `wasm-tools` workload must be installed (`dotnet workload install wasm-tools`) — without it, ComputeWasmBuildAssets fails with MSB4216
- Build with sandbox disabled for NuGet restore (network access required)
- Project source lives in `src/TurdTracker/`
- MudBlazor v9.2.0 configured with providers in MainLayout.razor
- Blazored.LocalStorage v4.5.0 (must specify version explicitly; latest search fails on .NET 10)
- Folder structure: Pages/, Components/, Models/, Services/ under src/TurdTracker/
- MudBlazor CSS/JS referenced in wwwroot/index.html
- Bootstrap CSS removed from index.html (MudBlazor replaces it)
- Build from repo root: `DOTNET_CLI_USE_MSBUILD_SERVER=0 dotnet build src/TurdTracker/TurdTracker.csproj`
- DiaryService uses LocalStorage key "diary-entries"
- DI registration pattern: `builder.Services.AddScoped<IService, Service>()` in Program.cs
- Page injection pattern: `@using TurdTracker.Services` + `@inject IDiaryService DiaryService`
- MudDatePicker binds to `DateTime?`, MudTimePicker binds to `TimeSpan?`
- Two-way binding for custom components: `@bind-Foo` requires `Foo` parameter + `FooChanged` EventCallback
- MudChip `OnClose` renders a removable chip; `OnClick` makes a clickable chip
- MudBlazor v9: use `ShowMessageBoxAsync` (not `ShowMessageBox`) on `IDialogService` for confirmation dialogs
- Navigation layout: MudDrawer + MudNavMenu for desktop side nav; custom bottom nav with MudToolBar for mobile
- Responsive breakpoint: 960px (mobile below, desktop above) using CSS media queries with `.desktop-only` / `.mobile-only` classes
- FAB bottom position should be 80px (not 24px) to clear the mobile bottom nav bar
- Group entries by date: `DateOnly.FromDateTime(e.Timestamp)` with Dictionary for O(1) lookup
- Monday-first calendar offset: `((int)dayOfWeek + 6) % 7`
- ThemeService uses LocalStorage key "theme-dark-mode" for dark/light mode persistence
- MudThemeProvider `@bind-IsDarkMode` + `Theme` prop controls dark/light mode switching
- MudBlazor v9 MUD0002 warning: avoid HTML attributes like `Title` on MudBlazor components; use MudBlazor-specific props
- `_Imports.razor` has global `@using TurdTracker.Services` — no per-page using needed
- MudBlazor v9 charts: `MudChart<double>`, `ChartSeries<double>`, `BarChartOptions` (concrete, not `ChartOptions`). Labels prop is `ChartLabels`. Need `@using MudBlazor.Charts` and `@using TurdTracker.Models`.
- MudBlazor v9: Donut/Pie chart `InputData`/`InputLabels` props trigger MUD0002 warnings — prefer Bar chart with `ChartSeries` API for all chart types.
- PDF export: use `window.print()` via JS interop with `@media print` CSS to hide app chrome and show print-only content
