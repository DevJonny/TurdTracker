## Codebase Patterns
- Build command: `DOTNET_CLI_USE_MSBUILD_SERVER=0 DOTNET_HOST_PATH=$(which dotnet) dotnet build` (required on macOS ARM64 to avoid task host error)
- `wasm-tools` workload must be installed (`dotnet workload install wasm-tools`) ŌĆö without it, ComputeWasmBuildAssets fails with MSB4216
- Build with sandbox disabled for NuGet restore (network access required)
- Project source lives in `src/TurdTracker/`
- MudBlazor v9.2.0 configured with providers in MainLayout.razor
- Blazored.LocalStorage v4.5.0 (must specify version explicitly; latest search fails on .NET 10)
- Folder structure: Pages/, Components/, Models/, Services/ under src/TurdTracker/
- Build from repo root: `DOTNET_CLI_USE_MSBUILD_SERVER=0 dotnet build src/TurdTracker/TurdTracker.csproj`
- DI registration pattern: `builder.Services.AddScoped<IService, Service>()` in Program.cs
- `_Imports.razor` has global `@using TurdTracker.Services` ŌĆö no per-page using needed
- JS interop files live in `wwwroot/js/`; call from C# via `IJSRuntime.InvokeAsync("googleAuth.method", args)` (no `window.` prefix)
- GoogleAuthService has lazy init via `_initialized` flag and `EnsureInitializedAsync()` ŌĆö all public methods call it before JS interop
- GoogleAuthService wraps JS interop; inject `IGoogleAuthService` for auth operations
- SyncService.InitializeAsync() is called from MainLayout.OnAfterRenderAsync(firstRender) ŌĆö this is the app startup entry point for sync
- GIS token client uses `requestAccessToken()` for interactive sign-in; `requestAccessToken({ prompt: '' })` for silent (no popup)
- GIS silent sign-in fails if: Google session expired, consent revoked, or popup blocked ŌĆö handle gracefully by resolving null
